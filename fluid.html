Fixed width vs Fluid layouts

In this example there are two <div> elements. The first has a fixed width of 1000px. The second has a width of 90%. Try resizing the browser window. You'll see that the first block keeps its width fixed but that the second resizes in response to the width of the window.

On the desktop you're probably seeing the first, unresponsive block simply being clipped, or hidden as the browser window is resized. If you look on a mobile device you'll see something different. The unresponsive block is restricted to the full width of the browser window and the text wraps correctly. So mobile browsers attempt to give a better experience out of the box. Its not actually possible to resize a mobile or tablet browser (currently!) and so sensible default behaviour has been implemented meaning that we can view all of the text.

The responsive block looks better, it has correctly scaled to take up just 90% of the total browser width. This already gives us more consistency across devices.
